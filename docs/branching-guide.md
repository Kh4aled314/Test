# Branching Guide

## Recommended branch names

- `feature/add-search` ‚Äî new functionality
- `fix/typo-in-readme` ‚Äî bug fixes
- `docs/update-setup` ‚Äî documentation only
- `chore/cleanup` ‚Äî maintenance tasks

## Workflow

1. Create a branch from `main`
2. Make your changes
3. Commit with a clear message
4. Push the branch
5. Open a Pull Request on GitHub
6. Merge after review (even self-review counts for practice!)

## Practice conflict scenario

Edit line 3 of `src/tasks.js` on two different branches with different text, then merge them to practice resolving conflicts.
